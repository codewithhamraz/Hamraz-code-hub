1.1 What is a Module? (The Reusability Nodes)

Jab hamara program bada aur complex ho jata hai, toh pure code ko ek hi script file mein rakhna lagbhag impossible aur unmaintainable ho jata hai. Is problem ko solve karne ke liye hum modules ka use karte hain. Ek module ek aisi file hoti hai jisme pehle se likha hua Python code (functions, classes, variables) hota hai, jise hum apni kisi bhi nayi file mein import keyword ka use karke direct reuse kar sakte hain. Modules do prakar ke hote hain:

  • 1. Built-in Modules (Standard Libraries): Yeh Python installation ke sath system mein pehle se auto-configure hokar aate hain. Inhe chalane ke liye internet ya kisi external download ki zaroorat nahi hoti. Jaise ki:
    • os: Operating system ke directory aur files se interact karne ke liye.
    • math: Complex mathematical calculations aur trigonometry operations ke liye.
    • random: Cryptographic aur pseudo-random numbers generate karne ke liye.
    • sys: Python runtime environment aur system-specific parameters ko control karne ke liye.
  • 2. External Modules (Third-Party Libraries): Yeh modules Python standard community ke developers ne banaye hain aur yeh open-source internet repositories par available hote hain. Inhe pehle PIP ke zaroorat se download karna padta hai. Jaise:
    • pandas aur numpy: Heavy data science aur matrix mathematical calculations ke liye.
    • django aur flask: Enterprise web back-end architectures build karne ke liye.
    • cowsay: Terminal automation aur visual text manipulation ke liye.

1.2 PIP: Python Package Manager Engine

PIP ka full form hai "Pip Installs Packages". Yeh Python ka ek official command-line package management tool hai. Iska kaam internet par bane sabse bade open-source Python repository PyPI (Python Package Index) se connect hokar kisi bhi external package ko aapke local computer ya server par automatic download, install, aur configure karna hota hai.

Important Terminal Commands for PIP Architecture:

# Installing any external third-party module globally
pip install package_name

# Uninstalling an existing package from the system environment
pip uninstall package_name

# Listing all installed packages along with their exact versions
pip list

1.3 Comments and Documentation Strings (Docstrings)

Comments source code ke andarz likhe gaye engineer ke personal notes hote hain. Jab Python ka interpreter program ko chala raha hota hai, toh voh comments vaali lines ko poori tarah se ignore (bypass) kar deta hai. Comments ka use code ke logic ko asan bhasha mein samjhane aur bade software teams ke beech coordination banane ke liye kiya jata hai.

  • Single-line Comments: Yeh comments sirf ek single line ke hote hain aur inki shuruat hamesha # (hash) character se hoti hai.
  • Multi-line Comments / Docstrings: Jab hume bade algorithms ya poore software architecture ka description likhna hota hai, toh hum triple quotes """ ya ''' ka use karte hain. Inhe formal taur par documentation strings bhi kaha jata hai.

Chapter 2: Variables and Core Dynamic Data Types (Complete Deep Dive)

Variables programming architecture ke sabse mahatvapurna block hain. Computer ke andarz ek physical storage memory address (RAM) hota hai. Ek software engineer RAM ke un complex binary addresses ko yaad nahi rakh sakta, isliye hum un memory locations ko ek unique alphanumeric name de dete hain, jise Variable kehte hain. Variable programming execution pipeline ke run-time ke dauran kisi bhi data payload ya values ko temporary register hold karke rakhta hai.

⚠️ Core Architecture Rule: Variables constant nahi hote; inki values execution ke dauran kisi bhi time badli (mutate) ja sakta hain, isiliye inhe 'Variable' (badalne wala) kaha jata hai.

2.1 Python Dynamic Typing & Memory Allocation Mechanics

Baki static programming languages (jaise C++, Java, ya C#) ki tarah Python mein variable initialize karte waqt uski data type pehle se declare nahi karni padti (e.g., int price = 400; nahi likhna padta). Python ka internal execution engine runtime par variable ke andarz save data ke signature aur configuration ko automatically analyze karta hai, aur real-time mein use specific data type address memory logic assign kar deta hai. Is power feature ko **Dynamic Typing** kehte hain.

2.2 Primitive and Advanced Data Types Taxonomy

Python standard data architecture variables default roop se in data parameters par directly map hoten hain:

  • Integers (int): Yeh bina kisi precision matrix ya decimal points ke pure numbers hote hain. Inme positive aur negative dono ranges aati hain. Jaise: a = 45, b = -8921, c = 0.
  • Floating Point Values (float): Precision mapping numbers jisme data structural values mathematical decimals holding arrays ke sath dynamic perform hoti hain. Jaise: pi = 3.14159, gravity = -9.8.
  • Strings (str): Characters ka ordered sequence vector jo single quotes ('), double quotes ("), ya triple quotes (""") ke andarz variable blocks hold karte hain. Jaise: name = "Hamraz Tariq".
  • Booleans (bool): System logical conditional status switches jo program control flow ko evaluate karne ke liye sirf do absolute states use karte hain: True aur False.
  • None Type: Variable initialization authentication step validation condition jahan memory register placeholder create ho jata hai par uske andarz koi concrete value assign nahi hoti: state = None.

2.3 Rules for Constructing Valid Identifiers (Variables)

Python variables name conventions core strict rules guidelines ko bind karte hain:

  • Variable identifier name strictly kisi alphabet letter (A-Z, a-z) ya fir underscore (_) symbol character se hi start hona chahiye.
  • Variable name ka pehla character kabhi bhi koi numeric digit integer (0-9) nahi ho sakta. (e.g., 101nodes error dega, par nodes101 perfect chalega).
  • Variable ke beech mein space blank block restricted hai, space ki jagah hum underscore (_) se system word spacing structuralize karte hain (e.g., core_server_ip).
  • Python character layout variables completely case-sensitive parameters check karte hain. Iska matlab system_load aur SYSTEM_LOAD dono alag memory references registers locations represent karenge.
