1.1 What is a Module? (The Reusability Nodes)

Jab hamara program bada aur complex ho jata hai, toh pure code ko ek hi script file mein rakhna lagbhag impossible aur unmaintainable ho jata hai. Is problem ko solve karne ke liye hum modules ka use karte hain. Ek module ek aisi file hoti hai jisme pehle se likha hua Python code (functions, classes, variables) hota hai, jise hum apni kisi bhi nayi file mein import keyword ka use karke direct reuse kar sakte hain. Modules do prakar ke hote hain:

  • 1. Built-in Modules (Standard Libraries): Yeh Python installation ke sath system mein pehle se auto-configure hokar aate hain. Inhe chalane ke liye internet ya kisi external download ki zaroorat nahi hoti. Jaise ki:
    • os: Operating system ke directory aur files se interact karne ke liye.
    • math: Complex mathematical calculations aur trigonometry operations ke liye.
    • random: Cryptographic aur pseudo-random numbers generate karne ke liye.
    • sys: Python runtime environment aur system-specific parameters ko control karne ke liye.
  • 2. External Modules (Third-Party Libraries): Yeh modules Python standard community ke developers ne banaye hain aur yeh open-source internet repositories par available hote hain. Inhe pehle PIP ke zaroorat se download karna padta hai. Jaise:
    • pandas aur numpy: Heavy data science aur matrix mathematical calculations ke liye.
    • django aur flask: Enterprise web back-end architectures build karne ke liye.
    • cowsay: Terminal automation aur visual text manipulation ke liye.

1.2 PIP: Python Package Manager Engine

PIP ka full form hai "Pip Installs Packages". Yeh Python ka ek official command-line package management tool hai. Iska kaam internet par bane sabse bade open-source Python repository PyPI (Python Package Index) se connect hokar kisi bhi external package ko aapke local computer ya server par automatic download, install, aur configure karna hota hai.

Important Terminal Commands for PIP Architecture:

# Installing any external third-party module globally
pip install package_name

# Uninstalling an existing package from the system environment
pip uninstall package_name

# Listing all installed packages along with their exact versions
pip list

1.3 Comments and Documentation Strings (Docstrings)

Comments source code ke andarz likhe gaye engineer ke personal notes hote hain. Jab Python ka interpreter program ko chala raha hota hai, toh voh comments vaali lines ko poori tarah se ignore (bypass) kar deta hai. Comments ka use code ke logic ko asan bhasha mein samjhane aur bade software teams ke beech coordination banane ke liye kiya jata hai.

  • Single-line Comments: Yeh comments sirf ek single line ke hote hain aur inki shuruat hamesha # (hash) character se hoti hai.
  • Multi-line Comments / Docstrings: Jab hume bade algorithms ya poore software architecture ka description likhna hota hai, toh hum triple quotes """ ya ''' ka use karte hain. Inhe formal taur par documentation strings bhi kaha jata hai.
